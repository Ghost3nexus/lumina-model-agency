## 1. Vision

### プロダクト定義

- **一言で**: ファッションECの商品撮影を、プロの制作会社レベルのAI画像生成で置き換えるサービス
- **誰のため**: ファッションEC事業者（ブランド・セレクトショップ・D2C）。月30〜300SKUの商品回転を持つ中堅〜中小
- **何を解決するか**: 商品撮影にかかるコスト（月¥24〜70万）・時間（1SKU 2〜3時間）・品質のばらつき
- **使った後どうなるか**: 撮影コスト80〜90%削減。納品即日。品質が毎回一定。モデル確保の悩みゼロ

### なぜ品質が全てか（最重要）

ファッションECは**写真を売っている**のと同義。

- 顧客は実物を手に取れない。商品ページの写真が購買判断の全て
- 低品質な画像 = ブランド毀損 = 売上減少。だからEC事業者は撮影に月数十万かけている
- 「AIで安く作れます」だけでは売れない。**「AIで作ったと気づかないレベル」**で初めて検討対象になる
- ZARAがEC参入した時、写真のクオリティの高さで世界を騙した。写真で市場は動く
- スマホ撮影・素人品質・SNSエンタメ用途のAI画像は、ECの商品ページでは**絶対に使われない**

**品質が一線を超えれば、EC事業者は「どう考えても使う」。なぜなら撮影コストは誰もが削減したい最大のペインだから。**

### ポジショニング

**制作会社に認められる品質 × AIの速度とコスト = 唯一のポジション**

| | 制作会社 | botika | 汎用AIツール | **Lumina** |
|---|---|---|---|---|
| 品質 | 最高 | 高い | 低〜中 | **制作会社レベル** |
| コスト | 月¥24〜70万 | $15〜225/月 | 無料〜¥5,000 | **制作会社の1/5〜1/10** |
| 速度 | 1〜2週間 | 数分 | 即時 | **即日** |
| 日本市場対応 | 対応 | 非対応 | 一部 | **完全対応** |
| モール規格 | 手動対応 | 非対応 | 非対応 | **楽天/Yahoo/Amazon自動対応** |

### 競合（最重要参考: botika.com）
- botika: $15〜225/月、ファッション特化、人間レビューQA。テキストプロンプト非対応・日本語NG
- Photoroom: 背景ツール。ファッション特化度低い
- Claid.ai: 生地・ロゴ忠実度高い。日本語UI微妙
- WearView: 新興。ポーズ制御・モデルID一貫性
- 日本国内（ModaAI, AI model, アイファモ等）: 品質が制作会社レベルに未達。**ここが参入余地**

## 2. Quality Bar — 最重要セクション

### 品質判定者

**社長（KOZUKI）= プロのファッションディレクター**

社長が「これはECの商品ページに載せられる」と判断するまで、そのタスクは未完了。
自己判断で「完了」にしてはならない。

### ベンチマーク

| レベル | 定義 | 判定 |
|--------|------|------|
| **S（目標）** | 制作会社が撮影した写真と見分けがつかない | プロが「これAIなの？」と言う |
| **A（合格）** | ECの商品ページに掲載して違和感がない | 社長が「OK」と言う |
| **B（不合格）** | AIっぽさが残る、素材感が弱い | 社長が「まだダメ」と言う → 修正 |
| **C（論外）** | 白飛び、フラットライティング、不自然なポーズ | 提出するな |

**参考:**
- botika.com のギャラリー（最低ライン）
- ZARA EC商品ページ（理想ライン）
- NET-A-PORTER 商品ページ（ハイエンドの参考）

### 画像品質の絶対基準

**1. ライティング**
- 影比率: 最低1:2.5〜1:3（ソフトだが立体感のあるディレクショナルライティング）
- 白飛び禁止。ハイライトのディテールが必ず残っていること
- フラットライティング（影なし・全面均一）は**絶対禁止**
- 光源の方向が感じられること

**2. 素材感・テクスチャ**
- 服の素材（コットン・シルク・デニム・ニット等）が画像から判別できること
- 縫い目・ステッチ・織り目が見えるレベルの解像度
- 白い服でもテクスチャが消失しないこと（白飛びの最大の被害者）

**3. モデル**
- 不自然なポーズ禁止（ロボット直立、腕の角度が不自然等）
- 表情に生気があること（無表情・死んだ目は不合格）
- 体型のプロポーションが自然であること
- 服とモデルの境界が自然に馴染んでいること（VTON精度）

**4. 背景**
- EC用白背景: 純白(#FFFFFF)で商品が浮かない
- 各モール規格準拠（楽天・Yahoo・Amazon）
- エディトリアル背景: 服の世界観に合った背景選択

**5. 全体**
- 解像度: 最低1024x1024、推奨2048以上
- ファイルサイズ: JPEG品質0.93以上 or PNG
- 色味: 実物に忠実。過度な彩度補正禁止

### 品質チェック手順（実装時に必ず実行）

```
1. 実際に5枚以上生成する（白Tシャツ、黒ジャケット、柄物ワンピース、デニム、ニットの5種）
2. botika.comのギャラリー画像と並べて比較する
3. 以下をセルフチェック:
   □ 白飛びしていないか（ヒストグラムで255に張り付いていないか）
   □ 影にディテールがあるか（影の中の素材感が見えるか）
   □ 素材が判別できるか（コットンとシルクの違いがわかるか）
   □ モデルのポーズが自然か
   □ 服とモデルの境界が自然か
4. 1つでもNGがあれば修正してから報告する
5. 「動きました」は報告ではない。「botika比較でこのレベルです」が報告
```

## 3. Architecture

### 技術スタック
- **フロントエンド**: React 19 / TypeScript / Vite / Tailwind CSS
- **3D**: @react-three/fiber, @react-three/drei, three.js
- **AI画像生成**: Gemini 3 Pro Image Preview API（メイン）
- **仮想試着（VTON）**: IDM-VTON（ECCV 2024）via Replicate
  - フォールバック構成: FASHN(primary) → IDM-VTON(fallback) → Gemini(final)
- **背景除去**: lucataco/remove-bg (Replicate)
- **インフラ**: Vercel

### 本番情報
- **Vercelプロジェクト**: `llumina-fashion-studio`
- **本番URL**: https://llumina-fashion-studio.vercel.app/

### 重要な技術的制約
- Gemini画像生成1回: 約$0.02〜0.05
- Replicate IDM-VTON 1回: 約$0.05〜0.15
- 画像設定: MAX_IMAGE_BYTES=10MB, JPEG品質0.93
- プロンプトは短い方が高品質（1,200語→400語で改善確認済み 3/5）

## 5. Rejection Log（ダメ出し履歴）— 必読

> 社長のダメ出しは「品質基準の蓄積」。同じミスを繰り返すな。

### 2026-03-10 画像品質 — 白飛び問題
- **ダメ出し**: 「生成されてくる画像がずっと白飛び。クオリティがbotikaの半分にも行ってない」
- **原因**: EC STUDIO LIGHTINGブロックで「flat light quality」「nearly shadowless」「shadow ratio 1:1.5」を指定
- **修正**: 影比率を1:2.5〜1:3に変更。BLOWN_HIGHLIGHTS(-20点+ハードフェイル)、LIGHTING_DIMENSION(-15点)、FABRIC_TEXTURE(-10点)を品質チェックに追加
- **学習**:
  - ECプロダクト画像でフラットライティングは**絶対禁止**
  - 「動きました」ではなく「botika比でこのレベルです」が報告の最低ライン
  - 品質チェック関数を書いたら、実際の出力画像で検証してから完了報告する

### 2026-03-10 画像品質 — 機能偏重
- **ダメ出し**: 「Studio Preset、Brand Profileをなくしても良い。大事なのは生成された画像のクオリティだけ」
- **学習**:
  - 機能の数ではなく、出力の品質が全て
  - UIの設定項目が多くても、出力が悪ければ意味がない
  - 社長は「動くかどうか」ではなく「使えるかどうか」で判断する

### 2026-03-05 プロンプト設計
- **発見**: プロンプトは短い方が高品質（1,200語→400語で改善）
- **学習**: 長いプロンプト = 高品質ではない。AIに余計な制約を与えると品質が下がる

## 行動原則

1. **品質が全て** — 機能追加より出力品質。動くことより使えること
2. **botika比較が最低ライン** — 報告時は必ずベンチマーク比較を添付
3. **プロンプトは短く** — 余計な制約は品質を下げる
4. **Before/After必須** — 品質に影響する変更は比較画像なしにマージしない
5. **社長判断が完了基準** — 自己判断で「完了」にしない
