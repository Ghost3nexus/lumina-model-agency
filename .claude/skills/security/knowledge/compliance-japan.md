# 日本法規制・コンプライアンス対応

## 個人情報保護法（2026年改正予定）

### 主な改正ポイント
1. **委託先管理の強化**: 個人データの処理を委託する際の管理義務が厳格化
2. **漏洩報告の厳格化**: 個人データ漏洩時、3〜5日以内に個人情報保護委員会へ報告
3. **サプライチェーン評価制度**: 委託先・再委託先のセキュリティ評価が義務化方向
4. **AI利用の透明性**: AIによる自動処理に関する説明義務の議論

### TomorrowProofでの対応
| 要件 | 現状 | 対応 |
|------|------|------|
| プライバシーポリシー | 各サービスに掲載 | 定期更新 |
| 漏洩報告体制 | インシデント対応手順あり | 報告テンプレート準備 |
| 委託先管理 | API利用（Anthropic, Google等） | 利用規約の確認・記録 |
| データ最小化 | Discord会話ログの保持 | 保持期間ルール策定 |
| 同意取得 | サービス利用規約で対応 | 明示的同意フロー検討 |

### データフロー記録
```
ユーザー（Discord）
  → Discord Bot（Railway）
    → Anthropic Claude API（処理のみ、保持なし）
    → freee API（会計データ）
    → Google APIs（広告・カレンダー）
  → Dashboard（Vercel）
    → Express API（Railway）
```

## IPA 中小企業の情報セキュリティ対策ガイドライン

### 5つの基本原則
1. **OSやソフトウェアは常に最新の状態に**
   - 対応: npm audit weekly, Dependabot自動更新, Node.js LTSフォロー

2. **ウイルス対策ソフトを導入**
   - 対応: サーバレス/コンテナなので従来型は不要
   - 代替: Trivy（コンテナスキャン）、Semgrep（SAST）

3. **パスワードを強化**
   - 対応: API鍵は十分な長さ（32byte+）、環境変数で管理
   - GitHub/Railway/Vercel: 2FA有効

4. **共有設定を見直し**
   - 対応: CORS明示設定、APIトークン認証、ファイルパーミッション600

5. **脅威や攻撃の手口を知る**
   - 対応: WebSearchで最新脅威情報を定期収集
   - IPA「重要なセキュリティ情報」をウォッチ

## AI利用に関するガイドライン

### 総務省「AI事業者ガイドライン」（2024年版、2026年更新予定）
1. **透明性**: AI利用を明示（ブログはエージェント名を著者として表示）
2. **公平性**: AI出力にバイアスがないか定期チェック
3. **安全性**: 誤情報生成リスクの低減（ファクトチェック手順）
4. **プライバシー**: 個人データのAI処理に関する同意取得
5. **アカウンタビリティ**: 問題発生時の責任体制の明確化

### AIコンテンツの取り扱い
- AI生成コンテンツの著作権: 人間の「創作的寄与」がある場合は著作権が認められうる
- 免責表示: AI生成であることを適切に開示
- ファクトチェック: AI出力を盲信せず、重要情報は人間が確認

## データ保持・削除ポリシー

### 保持期間
| データ種別 | 保持期間 | 根拠 |
|-----------|---------|------|
| Discord会話ログ | 90日 | 運用上必要 |
| freee取引データ | 7年 | 法人税法 |
| Google広告データ | 2年 | 分析用 |
| APIリクエストログ | 30日 | デバッグ用 |
| エラーログ | 90日 | 障害対応用 |
| 個人情報 | 利用目的達成まで | 個人情報保護法 |

### 削除手順
1. データ特定（対象ユーザー・データ種別）
2. バックアップからも削除
3. 削除ログの記録
4. 削除完了の確認・報告
