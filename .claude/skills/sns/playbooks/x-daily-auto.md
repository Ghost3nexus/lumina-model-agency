# X日次自動運用プレイブック

> `/sns` 起動時、または毎朝の自動実行で、X投稿3-5本を生成する。
> バイラルトレンド調査 + note記事引用 + トレンド連動を組み合わせ、
> KOZUKIがコピペするだけの状態で `/content/sns/x-daily/YYYY-MM-DD.md` に出力する。

---

## 実行フロー（毎朝自動）

```
Step 1: バイラルトレンド調査（WebSearch）
  → AI/テック系の最新バズ、競合の伸びている投稿、トレンドハッシュタグ
  → 調査KW: 「AIエージェント」「AI起業」「Claude Code」「ソロプレナー」「生成AI 活用」

Step 2: note記事から引用候補を抽出
  → /content/note/ 以下の公開済み記事を読む
  → 数字フック、名言、ストーリー切り抜きを3候補抽出

Step 3: 投稿3-5本を生成
  → 投稿パターンをローテーション（下記参照）
  → 各投稿に画像指示 or 既存画像パスを添付
  → リプライ誘導文も生成（URL誘導はリプで）

Step 4: 出力
  → /content/sns/x-daily/YYYY-MM-DD.md に保存
  → KOZUKIコピペ用フォーマットで出力
```

## 投稿パターン（日替わりローテーション）

| 曜日 | パターン1 | パターン2 | パターン3 |
|------|----------|----------|----------|
| 月 | note公開告知 | ブログ連動 | 問いかけ型 |
| 火 | note引用（数字） | トレンド連動 | 哲学・考察 |
| 水 | note公開告知 | ブログ連動 | データ図解型 |
| 木 | note引用（名言） | セキュリティ時事 | 会話誘発型 |
| 金 | 週振り返り数字 | トレンド連動 | 問いかけ型 |
| 土 | note引用（ストーリー） | 海外AI事例 | 哲学・考察 |
| 日 | note引用（体験談） | ソロプレナー論 | 問いかけ型 |

## note引用ツイート生成ルール

### パターンA: 数字フック
```
記事から衝撃的な数字を抽出 → X投稿の冒頭に配置
例: 「タスク完了率3.2%」「22体中18体が完了ゼロ」「8事業を同時運営」

フォーマット:
[数字]。
[補足の1行]

[ストーリー2-3行]

[結論 or 問いかけ]
```

### パターンB: 名言抽出
```
記事から刺さるフレーズを引用
例: 「外に出ていないものは存在しないのと同じ」「仲間は、作れる」

フォーマット:
「[引用フレーズ]」

[背景の補足2-3行]

[自分の解釈 or 問いかけ]
```

### パターンC: ストーリー切り抜き
```
記事のストーリーラインを凝縮
例: 「1月に遺書を書いた → 2ヶ月後22体のAI社員」

フォーマット:
[Before] → [After]

[プロセスの概要]

[教訓 or 次への布石]
```

## リプライ誘導（全投稿共通）

本文にURLは絶対に貼らない（-50〜90%ペナルティ）。
代わりにリプライで自然に誘導:

```
パターン1: 「noteで詳しく書いてます。プロフのリンクから。」
パターン2: 「続きはnoteで → プロフのリンクから読めます」
パターン3: 「全文はブログで公開してます（プロフから）」
```

| 12:00-13:00 | ★★★★☆ | note引用 or トレンド連動 |
| 15:00 | ★★★☆☆ | サブ投稿（オプション） |
| 18:00-19:00 | ★★★★☆ | 問いかけ or 哲学系 |
| 20:00 | ★★★☆☆ | 夜のエンゲージメント狙い（オプション） |

## アルゴリズム最適化チェック（全投稿で確認）

- [ ] 画像 or 動画を付けているか（テキストのみ禁止）
- [ ] 冒頭15-30字で衝撃数字 or 逆説か
- [ ] 150-200字に収まっているか
- [ ] 断言しているか（「かもしれない」禁止）
- [ ] 返信を誘発する問いかけがあるか
- [ ] 外部リンクが本文にないか（リプで誘導）
- [ ] ハッシュタグ2-3個（多すぎない）
