## PASONAの法則（新版）

| ステップ | 名称 | 役割 |
|---------|------|------|
| **P** | Problem（問題） | 読者の悩み・欲求を明確にする |
| **A** | Affinity（親近感） | 共感を誘い、ストーリーで距離を縮める |
| **S** | Solution（解決策） | 具体的な解決方法 + 根拠を提示 |
| **O** | Offer（提案） | 解決策を導入するための提案（無料お試し等） |
| **N** | Narrowing Down（絞込） | 期間・人数を絞り、今すぐ行動させる |
| **A** | Action（行動） | 明確な指示で行動を喚起する |

## TomorrowProof note記事テンプレート

### 構成（無料note — リード獲得用）

```markdown
# [タイトル: 衝撃数字 or 逆説]

## ▼ こんな悩みありませんか？（P: Problem）

- [ターゲットの悩み①]
- [ターゲットの悩み②]
- [ターゲットの悩み③]

👆 1つでも当てはまるなら、この記事は「保存必須」です。

---

## ▼ その悩み、放置するとどうなるか（A: Affinity — 煽り+親近感）

[問題を放置した場合の最悪シナリオを生々しく描写]

実は、僕自身も同じ状態でした。
[自分の体験談: Before の具体的な状況]

---

## ▼ 解決策はこれだった（S: Solution）

[具体的な解決方法を3-5ステップで解説]

### ステップ1: [具体的アクション]
[詳細説明 + データや根拠]

### ステップ2: [具体的アクション]
[詳細説明 + 実例]

### ステップ3: [具体的アクション]
[詳細説明 + Before→After]

**実際の結果:**
[具体的な数字・成果]

---

## ▼ さらに詳しく知りたい方へ（O: Offer）

[次のステップへの提案]
- 無料相談 / LINE登録 / ブログの完全ガイド

---

## ▼ ただし、今だけです（N: Narrowing Down）

[限定性・緊急性を提示]
- 「この記事を読んだ人限定で〇〇」
- 「3月中に行動した人だけが〇〇」

---

## ▼ 今すぐやること（A: Action）

**1. [具体的な行動指示]**
**2. [具体的な行動指示]**
**3. [具体的な行動指示]**

→ [CTA: プロフのリンク / LINE登録 / ブログURL]
```

## Affinity（親近感）の書き方ルール

1. **生々しく描写する** — 抽象的ではなく具体的な状況を書く
2. **自分の失敗体験を入れる** — 「実は僕も同じだった」
3. **Before→Afterを明確に** — 変化の落差が共感を生む
4. **ターゲットと似た人の声を使う** — noteは大衆向け、素人でもできた感
5. **理想の未来を具体的に描写** — 「これを手に入れたらどうなるか」

### TomorrowProof版 Affinity テンプレート
```
正直に言います。

2025年、僕は最悪の状態でした。
事業に失敗し、貯金はほぼゼロ。一緒にやってた仲間もいなくなった。

残ったのはMacBook1台だけ。

でも、そこからAIエージェント19体を構築して、
1人で4事業を回せるようになった。

あの時「もう無理だ」と思った自分に言いたい。
「大丈夫。武器は目の前にある」と。
```

---

## Solution の書き方ルール

1. **根拠を必ず示す** — データ、実験結果、お客様の声
2. **損失回避で訴求** — 「今やらないと〇〇損する」が最強
3. **具体的なステップで書く** — 「何をどうすればいいか」が明確
4. **権威性を添える** — 実績数字、大企業との取引、メディア掲載

## noteのアルゴリズム最適化（2026年版）

### おすすめ表示に載るには
- **スキ数** × **読了率** × **滞在時間** が重要
- 冒頭のフックで離脱を防ぐ（最初の3秒=Problem）
- 途中に画像・図解を入れてスクロール率UP
- 末尾のCTAでスキ誘導（「参考になったらスキお願いします」）

### SEO対策
- タイトルにメインKWを入れる（前半30字以内）
- 見出しにサブKWを入れる
- 本文2000字以上
- 内部リンク（自分の他のnote記事へ）

### 有料note vs 無料note
| 用途 | 無料 | 有料 |
|------|------|------|
| リード獲得 | ★★★★★ | ★★☆☆☆ |
| 収益化 | ☆☆☆☆☆ | ★★★★★ |
| SEO | ★★★★★ | ★★☆☆☆ |
| 拡散性 | ★★★★★ | ★☆☆☆☆ |

**TomorrowProof戦略**: 無料noteでリード獲得 → LINE/メルマガ登録 → 高単価商品販売

## チェックリスト（note記事公開前）

### 記事本文
- [ ] タイトルに数字 or 逆説が入っているか
- [ ] Problem で読者の悩みを3つ以上列挙しているか
- [ ] Affinity で自分の体験談を生々しく書いているか
- [ ] Solution に具体的な根拠（データ/声）があるか
- [ ] Narrowing Down で限定性・緊急性を入れているか
- [ ] Action で「今すぐ〇〇してください」と明確に指示しているか
- [ ] 本文2000字以上あるか
- [ ] 画像/図解が3枚以上入っているか
- [ ] 末尾にスキ誘導があるか
- [ ] 関連noteへの内部リンクがあるか
- [ ] Markdown表は使わず、太字リスト形式に変換しているか（note表示崩れ防止）
- [ ] 箇条書きは `-` ではなく `・` を使っているか（note表示最適化）

### サムネイル（見出し画像）— 必須チェック
→ 詳細: `./knowledge/note-thumbnail-guide.md`
- [ ] **3層コピーが入っているか**: キャッチフレーズ + メインコピー + サブタイトル
- [ ] **メインコピーが30文字以内か**
- [ ] **数字が含まれているか**
- [ ] **背景が明るい or 高コントラストか**（HPのダークテーマ#050508禁止）
- [ ] **文字が画像面積の40%以上を占めているか**
- [ ] **フォントが太いゴシック体か**（細いフォント禁止）
- [ ] **スマホの小さいサムネ表示でもコピーが読めるか**
- [ ] **1280x670px（1.91:1）で出力されているか**
