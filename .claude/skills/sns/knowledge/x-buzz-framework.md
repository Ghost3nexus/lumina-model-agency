# X(Twitter) バズ投稿フレームワーク

> Source: @xboost_official 53,568件のデータ分析（2026-03）
> KOZUKIが学習指示。全X投稿でこのフレームワークを適用すること。

---

## 核心データ

- **79%の投稿はRTゼロ**で消える。5件中4件は誰にも広まらない
- **6RT超えが分岐点**: RT1-5 → RT6-20 で中央値インプが7.8倍（1,780 → 13,799）
- **初速の6RTを取れるかが全て**。アルゴリズムの推薦フィード閾値
- **画像/動画あり = 4.4倍**のインプレッション

## フォロワー獲得の真実

- 「いいね」よりも**「ブックマーク」がフォロワーを増やす**
- BM21件以上 = BM0と比べて**フォロワー獲得196倍、インプ270倍**
- BM/いいね比率33%以上の投稿がフォロワー急増を引き起こす
- KPI = **いいね率ではなくBM率**
- 「面白い」より**「役に立つ」**がフォロワーを増やす
- 「これ、保存したくなるか？」を投稿前に自問する

## バズの2系統

### 系統A: 拡散爆発型（インプ・RT最大化）

AI・テック・社会観察・動画コンテンツに多い。

| # | パターン | 構造 | 使い方 |
|---|---------|------|--------|
| A1 | 個人体験 x 社会観察 | 自分の体験 → 社会的な問いに接続 | 「これって〇〇の問題でもあるよな」で締める |
| A2 | 数字でスケール | 想像を超える具体的数字を冒頭に | 「3年で〇〇万」「全体の79%が〇〇」 |
| A3 | 意外性・ギャップ | 期待 → 裏切りの落差 | 「〇〇は最強だと思ってたけど、実際使ってみたら...」 |
| A4 | 動画・画像のインパクト | テキストで伝わらないビジュアル | 必ず画像/動画を付ける。テキスト単体はNG |

### 系統B: 共感爆発型（いいね率・BM最大化）

恋愛・人間関係・哲学に多い。平均いいね率1.8%に対し6-8%台。

| # | パターン | 構造 | 使い方 |
|---|---------|------|--------|
| B1 | 本質を断言 | 「〇〇の本質は〇〇だ」 | 「かもしれない」→「〇〇だ」に変える |
| B2 | 逆説・反常識 | 「普通はAだが、実はB」 | 書く内容の「逆」から入る |
| B3 | 「自分のことだ」描写 | ターゲットの具体的あるある | 「みんな」ではなく具体的な状況描写 |

## 黄金テンプレート

```
【フック】衝撃的な事実 or 逆説（冒頭15〜30字）
  ↓
【根拠】具体例・数字（中盤）
  ↓
【結論】読者への問いかけ（末尾）

合計: 150〜200字 + 動画 or 画像
```

### テンプレ実例 — 拡散爆発型

```
53,568件のX投稿を分析した結果、79%がRTゼロで消えていた。

残り21%に共通していたのは「最初の6RTを超えるかどうか」。
6RTを超えた投稿は中央値インプが7.8倍に跳ね上がる。

つまりバズは運じゃなくて、初速の設計で決まる。
```

### テンプレ実例 — 共感爆発型

```
「毎日投稿すればフォロワー増える」は嘘。

53,568件分析した結果、フォロワーが増える投稿の共通点は
「いいねが多い」じゃなくて「ブックマークが多い」だった。

面白いだけじゃ人はフォローしない。
「保存したい」と思わせた瞬間にフォローされる。
```

## テーマ別戦略

| 目的 | 最適テーマ |
|------|----------|
| フォロワー増 | 投資・起業体験、ハウツー・Tips |
| インプ稼ぎ | AI・テック + 動画 |

## TomorrowProof投稿への適用ルール

1. **全投稿に画像を付ける**（x-cardプリセット 1200x675）
2. **冒頭15-30字で衝撃数字 or 逆説**を置く
3. **150-200字**に収める（280字上限だが短い方がバズる）
4. **BM狙い**: ハウツー・チートシート・具体的数字の分析系
5. **初速6RT**を取るため: 投稿直後にnote/ブログでシェア、コミュニティ活用
6. 投稿前チェック: 「これ、保存したくなるか？」「逆説か断言が入っているか？」「画像はあるか？」
7. **「〇〇かもしれない」は禁止**。断言する。
