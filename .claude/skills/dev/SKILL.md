## 最新情報の自動調査（必ず守る）

**開発に関わる質問・実装を行う前に、必ず公式ソースで最新情報を確認する。**
古い知識で実装すると、非推奨API・廃止機能・価格変更を見落とすリスクがある。
WebSearch / WebFetch を使い、公式ドキュメント・公式ブログ・公式SNSから最新情報を取得すること。

→ 公式ソース一覧は .claude/rules/tech-stack.md を参照

### チェックタイミング
- **開発着手前**: 使用するAPI・フレームワークの最新バージョン・Breaking Changesを確認
- **モデル選定時**: 最新のモデルID・価格・レート制限を公式から取得
- **デプロイ前**: ホスティングサービスの最新制限・料金を確認
- **エラー発生時**: 公式のステータスページ・既知の問題を確認

### 最新モデルID（定期更新が必要）
以下は参考値。**実装前に必ず公式で最新を確認すること。**

| プロバイダ | モデル | 用途 |

## AI APIコスト最適化

| 用途 | 推奨モデル | 理由 |
|------|----------|------|
| 長文生成（商品説明等） | Claude Sonnet | コスト効率が良い |
| 短文生成（タイトル等） | Claude Haiku | 最安・高速 |
| 複雑な推論・分析 | Claude Opus | 精度最優先の場合 |
| 画像生成 | OpenAI DALL-E / Stable Diffusion | 用途に応じて選択 |

## 開発原則

- **MVP優先**: 最小限の機能で素早くリリースし、フィードバックで改善
- **AI-First**: 可能な限りAI APIを活用して開発工数を削減
- **サーバーレス**: インフラ管理コストを最小化
- **ダークUIデフォルト**: 新規プロダクトはダークテーマを基本とする
- **モノレポ不採用**: 各プロダクトは独立したリポジトリで管理

## 行動原則

- **動くものを最速で**: 完璧な設計より、動くプロトタイプを優先
- **セキュリティ意識**: APIキー・個人情報の取り扱いは厳格に
- **コスト最適化**: 無料枠・サーバーレスを最大限活用。AI APIはモデル使い分け
- **ドキュメント**: 最低限のREADMEとコメントは必ず残す
- **再利用性**: 複数プロダクトで使い回せるコンポーネントは共通化する
- **Branding連携**: UIの世界観はBranding Agentの方針に従う
- **最新情報確認**: 実装前に必ず公式ドキュメントで最新API仕様・モデルID・価格を確認する。古い知識で実装しない
