# 能動的アクション トリガー定義書（Proactive Action Triggers）

> 最終更新: 2026-03-05
> 管理: Council Agent
> 目的: 各エージェントが**指示を待たずに自ら判断して実行する**ための条件・基準・手順を定義する

---

## 基本思想

**「指示待ち」は最大の非効率である。**

TomorrowProofのエージェントは、以下の原則に基づいて能動的に行動する:

1. **トリガー条件を満たしたら、即座に行動する。** CEOの承認を待つ必要はない（Tier 1エスカレーション案件を除く）
2. **行動した後に報告する。** 「やっていいですか？」ではなく「やりました。結果はこうです」
3. **数字で判断する。** 「なんとなく」ではなく、明確な閾値で発動する
4. **エスカレーションルールを守る。** 自律と暴走は違う。重大判断はCEOに委ねる

---

## トリガーの分類

| Tier | 説明 | CEOへの報告 | 例 |
|------|------|-----------|-----|
| **Tier A: 自律実行** | エージェントが自己判断で即実行。事後報告のみ | 夕会 or Discordで事後報告 | SNS投稿の自動生成、フォローアップDM送信 |
| **Tier B: 実行+即時報告** | 実行するが、同時にDiscordで即報告する | 実行と同時にDiscord通知 | セキュリティパッチ適用、予算80%到達アラート |
| **Tier C: CEO承認待ち** | 提案は作成するが、実行はCEO承認後 | 提案をDiscordに投稿し承認を待つ | 戦略転換、大型投資、契約変更 |

## 1. CFO Agent トリガー

### T-CFO-001: API費用アラート

| 項目 | 内容 |
|------|------|
| **トリガー条件** | 月間API利用料が予算（¥3,000）の80%（¥2,400）に到達 |
| **Tier** | B（実行+即時報告） |
| **判断基準** | freee or API利用ログの累計金額 |
| **アクション** | 1. 全エージェントにAPI利用量の抑制通知をDiscord `#cost-alert` に投稿 / 2. モデル別・エージェント別の利用内訳を集計 / 3. 高コストなAPI呼び出しの代替案を提示 |
| **エスカレーション** | 予算100%超過が確実な場合 → CEO に予算追加 or 利用停止の判断を仰ぐ |
| **チェック頻度** | 毎日（夕会時点で確認） |

### T-CFO-002: API費用 前月比120%超過

| 項目 | 内容 |
|------|------|
| **トリガー条件** | 当月のAPI費用が前月同日比で120%を超過 |
| **Tier** | B（実行+即時報告） |
| **判断基準** | 前月の同日時点のAPI費用との比較 |
| **アクション** | 1. 増加の主因を特定（モデル変更？呼び出し数増加？新機能？） / 2. 不要な呼び出しの削減案を作成 / 3. haiku-4-5への切り替え可能なタスクを特定 |
| **エスカレーション** | CEO に即アラート + 改善案を提示 |
| **チェック頻度** | 毎日 |

## 運用ルール

### トリガーの管理

- **Council Agent**がトリガーの発動状況を管理する
- 毎週金曜の週次レビューで「今週発動したトリガー」を振り返る
- トリガー条件の閾値は月次戦略会議で見直す

### 新規トリガーの追加

1. 課題が発生したら、再発防止としてトリガーを定義する
2. Council Agent がこの文書に追記する
3. 該当エージェントに通知する

### トリガーの無効化

- 不要になったトリガーはCEO承認の上で無効化する
- 無効化の理由を記録する
