# Council Daily Standup Playbook

## 目的
21エージェントの日次実行を確実にする。会議で決めたことを1日単位で消化する。

## 朝スタンドアップ（06:00）

### 1. 昨日の完了確認
```
各チームリーダーから報告:
- Revenue: 完了 [X/3] | 未完了理由: [...]
- Product: 完了 [X/3] | 未完了理由: [...]
- Strategy: 完了 [X/3] | 未完了理由: [...]
- Brand: 完了 [X/3] | 未完了理由: [...]
```

### 2. 今日のタスク確定（各チーム最大3件）
```
## Revenue チーム（今日）
1. [タスク] → [担当] | 完了基準: [...]
2. [タスク] → [担当] | 完了基準: [...]
3. [タスク] → [担当] | 完了基準: [...]

## Product チーム（今日）
1. [タスク] → [担当] | 完了基準: [...]
2. [タスク] → [担当] | 完了基準: [...]
3. [タスク] → [担当] | 完了基準: [...]

## Strategy チーム（今日）
1. [タスク] → [担当] | 完了基準: [...]
2. [タスク] → [担当] | 完了基準: [...]

## Brand チーム（今日）
1. [タスク] → [担当] | 完了基準: [...]
2. [タスク] → [担当] | 完了基準: [...]
```

### 3. KOZUKIタスク（5件以内）
```
## 今日のKOZUKIタスク（コピペ用）
1. [タスク] — [ファイルパス] — [所要時間]
2. [タスク] — [ファイルパス] — [所要時間]
合計: XX分
```

### 4. ブロッカー確認
- 他チームに依存しているタスクはないか
- KOZUKIの承認待ちはないか
- 外部サービスの問題はないか

## 夕方レビュー（18:00）

### 完了報告
```
## 本日の成果
- Revenue: [完了タスク] ✅ | [未完了タスク] ❌ 理由: [...]
- Product: [完了タスク] ✅ | [未完了タスク] ❌ 理由: [...]
- Strategy: [完了タスク] ✅ | [未完了タスク] ❌ 理由: [...]
- Brand: [完了タスク] ✅ | [未完了タスク] ❌ 理由: [...]

## 明日の仮タスク
（翌朝のスタンドアップで確定）
```

## 週次全体会議（金曜 18:00）

### アジェンダ
1. 週の完了率（目標: 80%以上）
2. 各チームKPI進捗
3. 翌週のタスク割当
4. ブロッカーの構造的解決
5. Plan B月次KPI進捗確認

## タスク状態管理

| 状態 | 定義 |
|------|------|
| pending | 未着手 |
| blocked | 依存タスク待ち |
| in_progress | 実行中 |
| completed | 完了 |
| deferred | 延期（新期限付き） |
| cancelled | 廃止 |
