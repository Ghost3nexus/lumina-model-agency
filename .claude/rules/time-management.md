# タイムマネジメントルール — 全エージェント共通

> 2026-03-09 CEO承認。Time Manager エージェントが横断管理。
> 全エージェントは自身のタスクに期限意識を持ち、遅延を未然に防ぐ。

---

## 原則

1. **期限のないタスクは存在しない** — 全タスクに期限を設定する
2. **20%バッファ必須** — 期限ギリギリのスケジュールは組まない
3. **遅延は早期申告** — 遅れそうな時点で即報告。完了後の報告は禁止
4. **KOZUKIの時間は最も希少なリソース** — 1日60分を超えない

---

## 全エージェント共通タイムルール

### タスク着手ルール
- タスク受領後 **30分以内に着手** or 着手予定時刻を返答
- 着手したら **進捗20%ごとに中間報告**
- 完了見込みが変わったら **即時にTime Managerへ通知**

### 時間見積もりルール
| タスク規模 | 見積もり | バッファ込み |
|---|---|---|
| 小（調査・確認） | 30分 | 36分 |
| 中（文書作成・分析） | 2時間 | 2.5時間 |
| 大（実装・複合タスク） | 半日 | 半日+1時間 |
| 特大（プロジェクト） | 1日 | 1日を複数タスクに分割 |

### デイリータイムライン（厳守）

```
06:00  Council朝会 → 全チームの今日タスク確定
08:30  News: Morning Briefing配信完了
09:00  Revenue: 営業文面・SNS投稿文をKOZUKIに納品
       KOZUKI: 営業コピペ送信 + X-01投稿
12:00  KOZUKI: X-02 + IG投稿
15:00  Google Ads: 日次消化率チェック
18:00  Council夕会 → 完了報告 + 翌日仮決定
       KOZUKI: X-03投稿 + Evening Report確認
```

## エージェント別タイムマネジメント付与

### Revenue チーム

**marketing**（コンテンツオーケストレーター）
- ブログ公開日（月/水）の **前日18:00** までにコンテンツパッケージ完成
- Writer → Visualizer → Dev の **リレー遅延を監視**
- 公開日当日09:00までにSNS投稿パッケージをSNSに納品

**sales**（営業）
- 営業メール/DM文面は **毎朝08:30** までに準備完了
- フォローアップは **前回接触から48時間以内**
- 商談日程は **3営業日前にリマインド**

**sns**（SNS運用）
- 投稿文は **投稿時刻の30分前** に準備完了
- スケジュール: 08:00 / 12:15 / 18:00
- 投稿後 **2時間以内にエンゲージメント初速チェック**

**google_ads**（広告運用）
- 日次予算消化率を **15:00にチェック**
- 異常消化（+30%/-30%）は **即時アラート**
- 月次レポートは **翌月3営業日以内**

**writer**（執筆）
- コンテンツパッケージ納品は **依頼から24時間以内**
- VISUALタグを含む完全版を納品（不完全版の仮納品禁止）

### Product チーム

**dev**（開発）
- スプリントタスクは **期限日の18:00完了**
- デプロイは **平日10:00-16:00のみ**（夜間・週末デプロイ禁止）
- 緊急バグは **検知から3時間以内に修正**

**security**（セキュリティ）
- 脆弱性発見から **重大: 24時間以内 / 中: 72時間以内 / 低: 1週間以内** に修正
- 定期監査は **月1回、月末の週に実施**

**visualizer**（ビジュアル）
- Writer納品後 **12時間以内に画像生成完了**
- OGP + Hero + SNS画像 = 最低5枚を一括納品

### Strategy チーム

**cfo**（経理・財務）
- 月次PL: **毎月25日に仮締め、月末確定**
- 請求書: **発行日から5営業日以内に送付**
- 支払い期限: **3営業日前にリマインド**

**tax**（税務）
- 確定申告: **期限60日前から週次チェック**
- 消費税: **期限45日前からカウントダウン**
- 決算対策: **期末3ヶ月前から開始**

**news**（ニュース）
- Morning Briefing: **毎朝08:30までに配信**
- 速報: **発生から1時間以内にアラート**

### Brand チーム

**legal**（法務）
- ローンチゲートチェック: **ローンチ7日前までに完了**
- 契約書レビュー: **依頼から48時間以内**
- 契約更新リマインド: **期限30日前**

## KOZUKI専用ルール

### 時間予算（1日60分）

| 活動 | 配分 | 上限 |
|---|---|---|
| 営業コピペ送信 | 15分 | 20分 |
| SNS投稿コピペ | 15分 | 20分 |
| 確認・承認・判断 | 15分 | 20分 |
| 予備・緊急対応 | 15分 | — |
| **合計** | **60分** | — |

### 時間超過時の対応

60分を超えた場合:
1. Time Manager が原因を特定
2. エージェントの準備不足なら → エージェント側で改善
3. タスク量の問題なら → 翌日に分散 or タスク削減
4. KOZUKI判断が必要な案件が多いなら → 判断基準を明文化してエージェントに委任
