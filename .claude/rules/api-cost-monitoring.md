## 異常検知ボーダーライン

### RED（即時CEO報告）

| 条件 | アクション |
|------|----------|
| Claude API **日次費用が日割り上限の200%超** | 即時報告 + 原因特定 + 高コストタスク停止検討 |
| Grok API **月次費用が予算100%到達** | 即時報告 + 利用停止判断 |
| **全体経費が月計の100%到達**（月途中で） | 即時報告 + 残日数分の支出凍結 |
| **単一タスクが$0.50（約¥75）超** | 即時報告 + モデル選定ミスの可能性 |
| **1エージェントが全体の50%以上を消費** | 即時報告 + 偏りの原因調査 |

### ORANGE（当日中にCEO報告）

| 条件 | アクション |
|------|----------|
| Claude API **月次費用が80%到達** | 残日数と消化ペースを計算 → 超過予測を報告 |
| Claude API **前日比300%以上の急増** | 原因特定（タスク増？モデル変更？ループ？） |
| **同一タスクの再実行が3回以上** | 無限ループ・エラーリトライの可能性 |
| Grok API **月次費用が80%到達** | 利用頻度の見直し提案 |

### YELLOW（週次レビューで報告）

| 条件 | アクション |
|------|----------|
| Claude API **月次費用が50%到達** | 消化ペースの記録 |
| **Opusモデルの使用が全体の20%超** | Sonnet/Haikuへのダウングレード検討 |
| **前月比120%超の増加傾向** | 増加原因の分析 |

## 日次チェックフロー

```
毎日 18:00（夕会時）:

1. costTracker.getDailyCosts() → 今日のClaude API費用を取得
2. 月初からの累計を算出
3. ボーダーラインと照合:
   - RED条件に該当 → 即時CEO報告（Discord #cost-alert）
   - ORANGE条件に該当 → 夕会で報告
   - YELLOW条件に該当 → 週次レビューに記録
4. 日割りペースで月末予測を算出
5. 超過予測がある場合 → 削減案を3つ提示
```

## コスト削減オプション（超過時の対策）

| 優先度 | 対策 | 想定削減率 |
|--------|------|----------|
| 1 | Sonnet → Haiku にダウングレード（短文タスク） | -60% |
| 2 | タスクのバッチ処理化（個別→まとめて） | -30% |
| 3 | キャッシュ導入（同じ質問の再実行を防止） | -20% |
| 4 | 低優先度タスクの延期（P3以下） | -40% |
| 5 | Opus使用を完全停止 | -80%（Opus分） |

## モデル別コスト意識（全エージェント必読）

| モデル | Input/1M | Output/1M | 1タスク平均 | 使用基準 |
|--------|----------|-----------|------------|---------|
| Haiku 4.5 | $1.0 | $5.0 | ~$0.01 | **デフォルト**。短文生成・分類・確認 |

| Opus 4.6 | $15.0 | $75.0 | ~$0.15 | **CEO承認時のみ**。複雑な推論・重要判断 |

> Opus 1回 ≒ Haiku 15回。コスト差を常に意識する。
