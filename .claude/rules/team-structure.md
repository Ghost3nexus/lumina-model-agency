# TomorrowProof 4チーム制組織構造

> 2026-03-09 Council決定。21エージェントを4チームに編成。
> 各チームにリーダーを置き、日次で完了報告を出す。
> **サブエージェント自律生成**: 全エージェントは負荷超過時にサブエージェントを自発的に生成できる（CEO承認済み）。
> → 詳細ルール: `.claude/rules/sub-agent-spawn.md`

---

## チーム編成

### Revenue チーム（売上直結）
- **リーダー**: marketing（コンテンツオーケストレーター）
- **メンバー**: sales, google_ads, sns, writer
- **ミッション**: 配信・営業・広告で売上を作る
- **日次KPI**: 配信本数、営業アプローチ数、リード数

### Product チーム（開発・保守）
- **リーダー**: dev（技術統括）
- **メンバー**: ui_ux_designer, security, visualizer
- **ミッション**: サービスを作る・守る・見せる
- **日次KPI**: 実装完了タスク数、バグ修正数

### Strategy チーム（戦略・管理）
- **リーダー**: planner（企画統括）
- **メンバー**: research, cfo, tax, news, ec_director
- **ミッション**: 調べる・計画する・管理する
- **日次KPI**: 調査レポート数、KPI更新

### Brand チーム（ブランド・法務・記録）
- **リーダー**: branding（ブランド統括）
- **メンバー**: document, legal, maruyama, journal
- **ミッション**: ブランドを守る・記録する・法務を固める
- **日次KPI**: 文書完成数、法務チェック完了数

### 横断ロール
- **council**: チーム横断ファシリテーター（日次スタンドアップ + 週次全体会議）
- **ceo**: 全チームの最終意思決定者

## 日次オペレーション

```
06:00  Council: 各チームの今日タスク確定（1チーム最大3件）
09:00  Revenue: 配信・営業実行 → KOZUKIコピペ（5件以内/55分以内）
10:00  Product: 開発・セキュリティ実行
14:00  Strategy: 調査・分析・レポート
16:00  Brand: 文書・法務・記録
18:00  Council: 完了報告 → 翌日タスク仮決定 → Journal記録
```

## タスク運用ルール

1. **1タスク = 1日完結**: 1日で終わらないタスクは分割する
2. **1エージェント = 1日最大3タスク**: それ以上は**サブエージェントを生成して委譲**する
3. **依存関係厳守**: 前段タスク完了まで後段タスクは「blocked」
4. **P3超過 → 延期 or 廃止**: P3で期限超過したタスクは即トリアージ
5. **KOZUKI手動 = 5件/日以内**: 合計60分以内。それ以上はエージェントが準備不足
6. **サブエージェント自律生成**: 負荷超過時、親エージェントの判断でサブを即時生成（承認不要、事後Journal記録）

### サブエージェント運用サマリー
- **命名**: `{親}:{役割}` （例: dev:integrator, security:auditor）
- **上限**: 1親あたり同時3サブ / 全体で同時10サブ
- **寿命**: 最大3日。超過は正式エージェント昇格をCouncil検討
- **権限**: 親と同じツール権限を継承。破壊的操作は親承認必須
- **品質**: サブの成果物は親が必ずレビュー
- **禁止**: サブがサブを生成すること（2階層まで）
- → 詳細: `.claude/rules/sub-agent-spawn.md`

## チーム間連携パターン

```
Revenue → Product: 「LPのここを直して」→ ui_ux設計 → dev実装
Product → Revenue: 「新機能できた」→ marketing企画 → writer記事 → sns投稿
Strategy → Revenue: 「この市場が熱い」→ sales営業リスト → marketing LP
Brand → Product: 「法的にここがダメ」→ legal指摘 → dev修正
```

## エスカレーションルール

- チーム内で解決できない → チームリーダーがCouncilに報告
- 2チーム以上に跨がる → Council会議を開催
- 売上/法務に直結 → CEO即判断
