# ファクトチェックフロー — X一次情報の検証ルール

> Council決定 2026-03-09。Grok + X API導入に伴い、全エージェント必読。

---

## 原則

**Xは一次情報の最前線だが、未検証情報が混在する。**
事業判断・コンテンツ制作・営業に使う前に、必ずファクトチェックを通す。

---

## 3段階信頼度ラベル

| ラベル | 定義 | 使用可否 |
|--------|------|---------|
| **A（確認済み）** | 公式ソース（公式サイト・プレスリリース・IR・公式ブログ）で裏取り完了 | ブログ・提案書・営業に使用OK |
| **B（蓋然性高）** | X上で3つ以上の独立ソースが一致、ただし公式未確認 | Morning Briefingで共有OK、外部発信は「〜と見られる」表現 |
| **C（要追加調査）** | 単一ソースのみ、噂段階 | 内部メモのみ。外部発信禁止 |

---

## フロー

```
1. X API (x_search) → ツイート取得（キーワード検索）
   ↓
2. Grok要約 → トレンドの概要・主要主張を抽出
   ↓
3. WebSearch → 公式ソースでクロスチェック
   ↓
4. ラベル付与（A / B / C）
   ↓
5. 保存（ソースURL + タイムスタンプ + ラベル必須）
```

---

## 保存形式

```markdown
### [トピック名]
- **ラベル**: A / B / C
- **概要**: [1-2文で要約]
- **X一次ソース**: [ツイートURL] ([投稿者] / [日時])
- **公式ソース**: [URL]（Aの場合必須）
- **取得日時**: YYYY-MM-DD HH:MM JST
```

---

## 適用エージェント

| エージェント | 適用場面 |
|------------|---------|
| **research** | 市場調査・競合分析時のX情報活用 |
| **news** | Morning Briefing・速報配信 |
| **sns** | X投稿のネタ元・リポスト判断 |
| **marketing** | コンテンツ企画のトレンド把握 |
| **sales** | 営業トークの業界動向ネタ |
| **writer** | 記事執筆時の最新事例引用 |

---

## 禁止事項

1. ラベルCの情報を外部発信に使ってはならない
2. X上の個人攻撃・噂話を事業判断の根拠にしてはならない
3. ソースURLなしで情報を保存してはならない
4. 1週間以上前のX情報は再検索して鮮度を確認すること
