## 0. Executive Summary

- **Category is contested, not empty** — "AI Model Agency" の EN/JP SERP には既存プレイヤーが存在(EN: Deep Agency, aimodelagency.com / JP: ai-model.jp, ai-admakers.jp)。当初の「世界で空白」仮説は**部分的に間違い**。
- **LUMINA の勝ち筋は "premium × character IP"**。ai-admakers は¥10,000/年の低価格戦略、LUMINA は¥5,000/月。**直接価格競合しない。Character Bible × IP tier × Ethics Code** の3柱でプレミアム領域を占有。
- "The world's first" 系の強気コピーは**使わない**(Deep Agency が既に同等を名乗っている)。代わりに「**The first AI Model Agency with Character Bibles**」「**Premium AI Model Agency**」で差別化。
- 90日目標: 英"AI fashion model agency"で top 10、日「AIモデル プレミアム」系 long-tail で top 5、自社ブランド名(LUMINA MODEL AGENCY)で top 1。

## 2. LUMINA の SEO ポジショニング(再調整)

### ❌ やらない
- `"AI model agency"` 素の語での全面戦争(ノイジーSERP、コモディティ競合)
- `"cheap AI model"` 系の価格訴求SEO(ai-admakersに絶対勝てない)
- 「世界初」のコピー(Deep Agency 等が同等主張、差別化にならない)

### ✅ やる
- **Premium positioning SEO**: 「プレミアム」「キャラクター」「IP」「ブランド専属」「ラグジュアリー」「Editorial」「Campaign」系キーワードを占有
- **Character Bible 訴求**: 競合にない独自概念を公式用語として定着させる(e.g. "Character Bible-driven AI models")
- **Long-tail × Vertical**: "AI model for luxury brand" "AI fashion model for Shopify" "AIモデル 専属契約" "AIモデル ブランド" 等
- **Editorial content** で権威性(E-E-A-T)構築。ジャーナル寄稿・メディア掲載・GitHub活動

- **H2**: サブラベル(Signature / New Faces / Curve etc.)

## 5. Technical SEO

### 5-1. Core Web Vitals 目標 (Lighthouse 90+)

| 項目 | 目標 |
|---|---|
| LCP (Largest Contentful Paint) | < 2.0s |
| INP (Interaction to Next Paint) | < 200ms |
| CLS (Cumulative Layout Shift) | < 0.1 |
| TTFB | < 0.6s |

**実装**: 
- 画像: `loading="lazy"` + `fetchpriority="high"` for Hero only
- Hero画像は `<link rel="preload" as="image">`
- Tailwind CDN → ビルドタイム埋め込みに移行(既存TODO)
- Font `Inter` は `font-display: optional`

### 5-2. Meta / Open Graph / Twitter Card

全ページ標準:
```html
<meta name="description" content="{page-specific}">
<meta property="og:type" content="website">
<meta property="og:title" content="{title}">
<meta property="og:description" content="{description}">
<meta property="og:image" content="{page-specific-ogp-1200x630.png}">
<meta property="og:url" content="{canonical}">
<meta property="og:site_name" content="LUMINA MODEL AGENCY">
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:site" content="@luminamodels">
<link rel="canonical" href="{canonical}">
```

OGP画像の命名: `/og/{page-slug}.png` (例: `/og/for-brands.png`, `/og/lucas-mori.png`)

### 5-3. Sitemap / robots

- `/sitemap.xml` 自動生成(React Router Static)
- `/robots.txt`:
  ```
  User-agent: *
  Allow: /
  Disallow: /studio/
  Disallow: /test/
  Sitemap: https://lumina-model-agency.vercel.app/sitemap.xml
  ```
- `/studio` をSEO対象外に(Council論点3: STUDIO非表看板方針)

### 5-4. hreflang (EN/JP bilingual)

Phase 2 対応。同一コンテンツのEN/JP版を `<link rel="alternate" hreflang="...">` で相互参照。

### 5-5. Schema.org 必須マークアップ

- `Organization` (全ページ共通、Navで)
- `BreadcrumbList` (すべての深いページ)
- `FAQPage` (/for-brands, /ethics)
- `CollectionPage` + `ItemList` (Division Pages)
- `Person` + `additionalType: FictionalCharacter` (/models/[slug])
- `Article` + `dateModified` (/ethics)
- `Service` + `Offer` (/for-brands)
